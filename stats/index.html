mythicpl.us: three months later

It feels like it's been longer, but the site has been up for three months. I figured now would be a good time to share some fun technical stats with you about webservers, analytics, and whatever other stuff I feel like talking about. Thanks for all your support so far.

Analytics

I brought the site up on November 28, 2016 after buying the mythicpl.us domain name for 88 cents during a Black Friday. My intent was to have a site where mythic plus information could be aggregated and edited based on player feedback. Speaking of which...

  • 191

    Total number of Github commits as of March 1, 2017. That's 191 total edits to the page or around 2 edits a day for 93 days (November 28 to Februrary 1, the time of writing).

  • 21

    Number of issues filed on Github to address incorrect data or issues with the site. 19 of those issues have been closed.

  • 7

    Pull requests submitted. 3 of those are from Gnarfoz fixing table information. 2 of those were site restylings.

  • 4

    Number of credited contributers.

User input is really important in a game where strategies or ways to do things change every week. The goal with having everything visible on Github was to allow players to drop in their thoughts, ideas, or corrections easily while having a visible record of things to do and enforcing accountability.

  • 370,671

    Total pageviews since online. Repeated views of a single page are counted.

  • 282,800

    Total sessions counted. A session is a time where someone is engaged with a site (e.g. clicks a link, etc). Of those sessions 54.5% were return users!

  • 128,685

    Unique users as counted by Google.

  • 51 seconds

    Average time spent on a page. Either it's quick to find the information you want, or the page is garbage and you want to leave! (I hope it's the former though)

Of note: the majority of sessions in November came from launch day (day one had over 5,000 concurrent sessions!). There was a considerable jump in traffic in January and Februrary that I attribute to end of holidays and rising in the Google search results.

Technical stuff

the server

The web server is a tiny ass Linux box running Fedora 24. It has 512MB of RAM (2GB in SWAP) and 1 CPU core. I'm basically using Github as a CRM for the web site. I write changes to the site, push them to Github and then have a cronjob set up to pull down the site changes from Github. I'm pretty lazy like that (i say as i haven't written a script to just automate everything).
